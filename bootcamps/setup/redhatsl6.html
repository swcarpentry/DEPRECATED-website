[The yum commands were tested using Scientific Linux release 6.2 (Carbon). I'm not sure what originally was on the server I tried this on.]

Scientific Linux 6 already comes with...

  • Bash shell
  • Make
  • vi text editor

All installation needs to be done as a root user or user with certain administrator privileges. If you have such access you can do

$ sudo su -

Then enter your usual password.

Install Nano, Emacs or XEmacs

Run

$ yum install nano
$ nano --version

Run

$ yum install emacs
$ emacs --version

Run

$ yum install xemacs
$ xemacs --version

Install Python

Run

$ yum install python

Install Python setuptools

Run

$ yum install python-setuptools

Install Python nose

Run

$ easy_install nose

And check

$ nosetests
----------------------------------------------------------------------
Ran 0 tests in 0.003s
OK

Install Mercurial

If using the latest version of EasyMercurial then you need at least Mercurial 1.7. Unfortunately the easy-to-use command

$ yum install mercurial

only installs 1.4. So we need to build from source!

Download the Mercurial source package from http://mercurial.selenic.com/.

This can be done at the command-line by copying the download link from the download page (using right-click and Copy Shortcut) then doing a wget e.g.

$ wget http://mercurial.selenic.com/release/mercurial-2.3.2.tar.gz

Beware that this link may have been updated since this page was written, so check the download page above for the URL to use.

Unpack the distribution

$ gunzip mercurial-2.3.2.tar.gz
$ tar -xf mercurial-2.3.2.tar
$ cd mercurial-2.3.2

Install dependencies

$ yum install python-devel
$ easy_install docutils

Now build

$ make install

Install EasyMercurial

Download the EasyMercurial Debian/Ubuntu source package from https://code.soundsoftware.ac.uk/projects/easyhg/files.
This can be done at the command-line by copying the download link from the download page (using right-click and Copy Shortcut) then doing a wget e.g.

$ wget https://code.soundsoftware.ac.uk/attachments/download/489/EasyMercurial-1.2.2-src.tar.gz

Beware that this link may have been updated since this page was written, so check the download page above for the URL to use.

Unpack the distribution

$ gunzip EasyMercurial-1.2.2-src.tar.gz
$ tar -xf EasyMercurial-1.2.2-src.tar
$ cd EasyMercurial-1.2.2-src

Install dependencies

$ yum install qt4
$ yum install qt4-devel
$ yum install PyQt4

Now build [The EasyMercurial README says to use qmake but this was not found so I used qmake-qt4]

$ qmake-qt4
$ make

Run

$ ./EasyMercurial

Install SQLite

SQLite does not need administrator privileges to run.

Download the SQLite Precompiled Binaries for Linux package from http://www.sqlite.org/download.html.

This can be done at the command-line by copying the download link from the download page (using right-click and Copy Shortcut) then doing a wget e.g.

$ wget http://www.sqlite.org/sqlite-shell-linux-x86-3071401.zip

Beware that this link may have been updated since this page was written, so check the download page above for the URL to use.

Unzip

$ unzip sqlite-shell-linux-x86-3071100.zip
$ ./sqlite3 --version

sqlite3 is the binary so you can put it and use it anywhere by anyone.

Install Firefox

Run

$ yum install firefox
